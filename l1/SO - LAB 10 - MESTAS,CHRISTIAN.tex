      \begin{enumerate}[nosep, leftmargin=\enumerateNestedMargin, series=questions]
        \tableContentsFontBold \item ¿Existen Thread en otros lenguajes de programación? \\
          \tableContentsFont Los threads existen en prácticamente todos los lenguajes de programación modernos, Python con threading, C++ con std::thread, C con POSIX threads (pthreads), JavaScript con worker threads, Go con goroutines, Rust con std::thread, etc.
        \tableContentsFontBold \item ¿Cuántos estados existen en el ciclo de vida de un Thread? \\
          \tableContentsFont Un thread pasa por estados similares a los de un proceso durante su ciclo de vida, NEW cuando el thread es creado pero no iniciado, RUNNABLE cuando está listo para ejecutarse o ejecutándose activamente, BLOCKED cuando espera adquirir algún lock, WAITING cuando espera a otro thread, TIMED\_WAITING cuando espera por un tiempo específico y TERMINATED cuando ha terminado su ejecución

% Conclusions and recommendations section
\noindent
\begin{tcolorbox}[tabletcolorbox={CONCLUSIONES Y RECOMENDACIONES}]
  \vspace{\tableCellPadding}
  \tableContentsFontBold
  \begin{enumerate}[nosep, leftmargin=\enumerateFirstMargin, series=conclusions]
    \item CONCLUSIONES: \\
      \tableContentsFont Java tiene dos formas de crear threads ya sea con Thread o con Runnable, donde Runnable es generalmente preferida por ser más flexible permitiendo que la clase herede de otras clases mientras mantiene la capacidad de ejecutarse en threads.
      
      \tableContentsFont La sincronización es muy importante para evitar problemas de concurrencia y se puede lograr mediante synchronized, wait, notify y join, estos mecanismos permiten coordinar la ejecución de múltiples threads controlando el acceso a recursos compartidos y creando dependencias entre threads.
  \end{enumerate}
  
  \tcbline
  
  \tableContentsFontBold
  \begin{enumerate}[nosep, leftmargin=\enumerateFirstMargin, resume=conclusions]
    \item RECOMENDACIONES: \\
      \tableContentsFont Se deben usar banderas volatile o AtomicBoolean para controlar el ciclo de vida de threads en lugar de usar métodos deprecados como stop, las banderas volatile garantizan que los cambios sean visibles inmediatamente entre threads, así el código es un poco más seguro y predecible.
      
      \tableContentsFont Cuando se trabaja con varios threads se debe usar estructuras de datos para gestionarlos en crear variables individuales, esto hace el código más mantenible, escalable y permite adaptar fácilmente el programa para crear diferentes cantidades de threads.
  \end{enumerate}
\end{tcolorbox}
