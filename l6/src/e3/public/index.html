Comparativa: RESTful vs GraphQL
Caracteristica RESTful (E1 — Spring Boot) RESTful (E2 — Flask) GraphQL (E3 — Yoga+Hono)
Endpoints Multiples: /api/books, /api/books/:id Multiples: /estudiantes, /estudiantes/:id Unico: /graphql
Metodos HTTP GET POST PUT DELETE GET POST PUT DELETE POST unicamente
Over-fetching Si — devuelve todo el objeto Si — devuelve todo el objeto No — solo los campos solicitados
Under-fetching Si — requiere multiples requests Si — requiere multiples requests No — una sola query con relaciones
Documentacion Externa (Swagger/OpenAPI) Externa (Swagger/OpenAPI) Integrada (introspection + playground)
Tipo de datos JSON (definido por el servidor) JSON (definido por el servidor) JSON (definido por el cliente)
Lenguaje Java Python TypeScript
Runtime JVM (Spring Boot) CPython (Flask) Bun (nativo JS)
Complejidad cliente Alta — multiples llamadas Alta — multiples llamadas Baja — una sola llamada
Caching Si — HTTP nativo Si — HTTP nativo No — requiere implementacion
